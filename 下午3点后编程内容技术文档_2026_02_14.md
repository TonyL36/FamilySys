下午3点后编程内容技术文档

一、背景与目标
下午3点后主要处理远亲关系查询的前端展示与关系准确性收敛，目标是：
1. 保持前端与远端页面行为一致，避免部署环境差异造成显示偏差。
2. 降低称谓输出复杂度，只在必要位置展示关系称谓。
3. 调整血缘紧密度模型，使代际跨度更贴近直觉。

二、实现内容
1. 结果展示调整
   - 成功查询时隐藏 result-summary，避免展示“称谓/描述”造成误解。
   - 查询失败或异常时继续显示提示信息，确保可用性。
2. 关系链路图展示
   - 恢复链路线上称谓文字，保证路径可读性。
   - 节点与连线保持 member1→member2 的方向一致。
3. 血缘紧密度模型
   - 计算公式调整为：φ = (1/2)^(血缘步数 - 1)。
   - 祖孙、外祖辈等关系按 2 步累计，体现代际跨度更大。
4. 远端页面清理
   - family-frontend 与 family-frontend-remote 逻辑同步。
   - 清理未使用的样式定义，减少冗余代码。

三、验证要点
1. 成功查询时 result-summary 不展示。
2. 关系链路上的称谓文字可见。
3. 血缘紧密度数值随代际变化而变化。
4. 本地与远端页面展示一致，仅 API 地址不同。

四、主要改动文件
1. family-frontend/distant-relative.html
2. family-frontend-remote/distant-relative.html

五、结论
下午3点后的修改聚焦在前端展示收敛与血缘紧密度模型的直觉一致性上，并完成了本地与远端页面同步与清理。
